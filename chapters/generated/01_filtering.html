2  02. Filtering

Run: run_02_optimized
Base de datos taxonómica: silva_138_2

2.1 1. Objetivo del bloque

Este bloque define el set de ASVs que alimenta los análisis downstream. El filtrado se aplica ahora a todas las muestras importadas: intestino, mocks y piensos/feed. Además, se exporta un subconjunto filtrado exclusivamente intestinal para los análisis biológicos principales.

El objetivo práctico es doble:

  • Retener una matriz global trazable donde mock y feed estén disponibles para controles técnicos, composición taxonómica y comparación con pienso.
  • Proteger los análisis biológicos de intestino exportando objetos ps_biological_*, sin mock ni feed.

2.2 2. Inputs, outputs y organización

2.2.1 2.1. Inputs principales

  • Objeto global importado: ps_all_silva_138_2.rds
  • Metadatos estandarizados: sample_metadata_standardized.csv
  • Parámetros: downstream_params.yaml

2.2.2 2.2. Tablas generadas

  • filtering_summary.csv: resumen por paso de filtrado.
  • filtering_summary_by_sample_type.csv: retención de lecturas por tipo de muestra.
  • prevalence_filter_groups.csv: grupos usados para calcular prevalencia.
  • sample_filtering_groups.csv: asignación de cada muestra a su grupo de prevalencia.
  • asv_filtering_status.csv: auditoría completa por ASV, con taxonomía, secuencia, reads y paso de eliminación.
  • removed_asvs.csv: ASVs eliminadas.
  • retained_asvs.csv: ASVs retenidas.
  • asv_counts_by_filter_step.csv: recuento de ASVs por motivo/paso de eliminación.
  • taxonomic_summary_by_filter_step.csv: resumen taxonómico de ASVs retenidas y eliminadas. multipestaña con las tablas principales.

2.2.3 2.3. Objetos exportados

  • Objetos globales filtrados: ps_final_silva_138_2.rds, ps_genus_silva_138_2.rds, ps_family_silva_138_2.rds y ps_phylum_silva_138_2.rds.
  • Objetos biológicos intestinales: ps_biological_final_silva_138_2.rds, ps_biological_genus_silva_138_2.rds, ps_biological_family_silva_138_2.rds y ps_biological_phylum_silva_138_2.rds.
  • Trayectoria completa: filtering_steps_silva_138_2.rds.

2.3 3. Diseño del filtrado

El filtrado se realiza a nivel de ASV. Primero se eliminan ASVs no deseadas por criterios taxonómicos, después se aplica un filtro de prevalencia dentro de grupos definidos por el diseño, y finalmente se retienen solo ASVs con abundancia relativa media global suficiente.

2.3.1 3.1. Filtro taxonómico

Se eliminan ASVs si cumplen alguno de estos criterios:

  • Asignación a Eukaryota o Unassigned.
  • Ausencia completa de asignación taxonómica en todos los rangos disponibles (Kingdom a Species).
  • Clasificación únicamente a nivel de reino, sin asignación por debajo.

No se eliminan ASVs asignadas a cloroplasto o mitocondria. Aunque este filtrado es habitual en análisis bacterianos estándar, aquí se conserva esa señal porque el pienso/feed contiene una fracción relevante de composición cloroplastidial o mitocondrial y esta información es necesaria para interpretar su relación con la microbiota intestinal.

2.3.2 3.2. Filtro de prevalencia

La prevalencia se calcula dentro de grupos biológicamente comparables:

  • Intestino: grupos diet_time, por ejemplo Ctrl_D07, BL15_D30.
  • Feed: Feed_Ctrl, Feed_BL15, Feed_BL30.
  • Mock: Mock.

La regla configurada es:

max(2 muestras, 10% del tamaño del grupo)

El umbral se limita al tamaño real del grupo. Esto permite que los piensos, que tienen una sola muestra por dieta, retengan ASVs presentes en esa muestra.

2.3.3 3.3. Filtro de abundancia media

Tras el filtrado de prevalencia, se retienen ASVs con abundancia relativa media global >= 0.0001 (0.01%).

2.4 4. Resultados principales

El dataset bruto contiene 138 muestras, 14,652 ASVs y 7,667,921 lecturas. Tras el filtrado final quedan 175 ASVs y 7,133,190 lecturas en el objeto global. El subconjunto intestinal final conserva 133 muestras, 169 ASVs y 6,931,390 lecturas.

2.4.3 4.3. Taxones relevantes eliminados

La tabla removed_asvs.csv permite revisar ASVs concretas que desaparecen. Entre las ASVs eliminadas por abundancia media hay ASVs asignadas a géneros como Photobacterium, Delftia, Companilactobacillus, Bacillus, Staphylococcus, Serratia o Bacteroides. Por ejemplo, ASV00214, asignada a Photobacterium, supera el filtro de prevalencia pero se elimina en el filtro final de abundancia media con 639 lecturas brutas.

Este punto es importante para la interpretación: que un taxón no aparezca en los plots finales no significa necesariamente que esté completamente ausente del dataset bruto; puede estar presente, pero no superar los criterios de prevalencia/abundancia definidos para el objeto final.

2.5 5. Interpretación metodológica

El filtrado produce un objeto final muy reducido en número de ASVs, pero con alta retención de lecturas. Este comportamiento es esperable en datos 16S: muchas ASVs aparecen con baja abundancia, baja prevalencia o asignación taxonómica débil, mientras que pocas ASVs concentran la mayor parte de la señal.

La decisión de aplicar el filtrado a todas las muestras mejora la trazabilidad: mock y feed ya no quedan fuera de la auditoría del filtrado. Aun así, los análisis biológicos de peces deben usar los objetos ps_biological_*, porque mock y feed tienen otra naturaleza experimental y no deben mezclarse con muestras intestinales en diversidad alfa, beta o abundancia diferencial.

El feed requiere una lectura específica. En la versión actual del filtrado, la señal de cloroplasto y mitocondria se conserva deliberadamente para no perder información composicional del pienso. Esto permite evaluar si parte de la señal observada en peces se aproxima al perfil del feed, pero obliga a distinguir entre señal microbiana bacteriana y señal derivada de ingredientes vegetales o material eucariota organelar.

2.6 6. Consecuencias para downstream

  • Los análisis de diversidad alfa, beta y abundancia diferencial deben partir de ps_biological_final o sus agregados taxonómicos.
  • Los análisis de composición que comparen intestino con feed pueden usar el objeto global filtrado, manteniendo claro que cloroplasto y mitocondria están conservados en esta versión.
  • Los análisis de mock deben interpretarse principalmente como control técnico de recuperación taxonómica y composición esperada, no como grupo biológico.
  • Para taxones concretos de interés, como Photobacterium, conviene consultar asv_filtering_status.csv antes de concluir ausencia/presencia en figuras finales.

2.7 7. Limitaciones y próximos pasos

El filtrado actual es adecuado para análisis globales conservadores, pero puede ocultar ASVs raras o transitorias. Si una pregunta se centra en colonizadores tempranos, pienso/feed, mock o taxones de baja abundancia, conviene realizar una revisión dirigida usando asv_filtering_status.csv o una versión de sensibilidad con umbrales menos estrictos.

Como próximos pasos, este bloque debería complementarse con:

  1. una figura QC de retención de reads por paso;
  2. una figura de distribución de prevalencia antes/después del filtrado;
  3. una revisión dirigida de ASVs eliminadas pertenecientes a taxones biológicos relevantes.

Estas figuras y estadísticas generales se recopilan en el bloque 03_dataset_stats.
